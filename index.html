CharFX Preroll Fix

VEX CharFX Simulation

When cloth preroll doesn't settle cleanly by shot start, this wrangle blends the two poses — no animator re-publish needed. TimeShift to the preroll frame on input 0, shot start on input 1, then plug both into this Point Wrangle.

Point Wrangle — preroll blend
vector P1 = point(0, "P", @ptnum);
vector P2 = point(1, "P", @ptnum);
int T1 = $RFSTART;
int T2 = $RFSTART + 100;
float shot_length = fit(@Frame, T1, T2, 0, 1);
float ramp = chramp("Translate", shot_length);

v@P = P1 + (P1 - P2) * ramp * -1;
Set the ramp to a B-Spline smooth transition. Bonus: this avoids loading the full heavy anim geo frame-by-frame during preroll — two birds, one stone.

Vellum Stiffness Ramping

VEX Simulation

Vellum stiffness operates on exponential values (10n), so remapping it linearly produces nothing useful. Remap the exponent instead. Source this outside DOPs for sanity.

Point/Detail Wrangle — stiffness ramp via exponent
i@frame_start = $FSTART;
i@frame_end   = $FEND;
f@stiffness   = 1;
f@exponent    = fit(@Frame, i@frame_start, i@frame_end, -10, 10);

f@remap = 1 * pow(10, @exponent);
Key insight: remap the exponent, not the stiffness directly. This gives a smooth, perceptually correct increase/decrease across any frame range.

CharFX Vector Deformation & Bend

VEX CharFX

When a shot starts with a collider already intersecting cloth and peaking outward isn't viable (complex extrusions, self-overlapping topology), split the region of interest, extract its centroid, and use this metaball-like push to separate the geo procedurally.

Point Wrangle — metaball push from centroid
// vector P1 = point(0, "P", 0);  // optional second centroid
vector P2   = point(1, "P", 0);
float min   = chf("minimum");
float max   = chf("maximum");
float ratio = chf("ratio");

float dist  = distance(v@P, P2);
float remap = fit(dist, min, max, 1, 0);
// float bend = chramp("bend", remap);  // use bend for custom falloff

v@P = v@P + (v@P - P2) * -(ratio) * remap;
Swap remap for chramp("bend", remap) to get a custom bend curve — effectively a hand-rolled Bend node that sidesteps the mysterious stretching Houdini's built-in sometimes produces.

Wrangle Blendshape

VEX

A minimal VEX blendshape with mask-driven blending. Point count must match between inputs. The float attribute acts as the blend weight, giving full per-point control.

Point Wrangle — masked blendshape
vector P1 = point(1, "P", @ptnum);
vector P2 = point(0, "P", @ptnum);
vector deltaP = P2 - P1;

float offset = ch("offset");
float foo     = clamp((f@float_variable + offset), -1, 1);
float scale   = chramp("ramppos", foo);

v@P = P1 + (scale * deltaP);
Drive float_variable from any attribute — painted weights, distance falloffs, or sim outputs — for maximum flexibility.

Vector Push & Direction Estimation

VEX

A simple point push, an averaged-normal technique for primitives, and a tangent-averaging trick to estimate the overall direction of a curve — useful for variably switching point and UV counts on curves.

Point Wrangle — push along vector
vector dir1 = set(0, 0, 0);
vector dir2 = set(1, 1, 1);
v@P += dir2 * chf("magnitude");
Detail Wrangle — average primitive normals
vector dir;

for (int i = 0; i < nprimitives(0); i++) {
    dir += prim(0, "N", i);
    dir  = normalize(dir);
}
Detail Wrangle — curve direction from tangents (Resample upstream first)
v[]@tangentu_list;
v@vector_sum;

for (int i = 0; i < @numpt; i++) {
    vector tangentu_pt = point(0, "tangentu", i);
    append(@tangentu_list, tangentu_pt);
}

@vector_sum = sum(@tangentu_list) / len(@tangentu_list);
Works reliably even for curves that change direction multiple times — the averaged tangent gives the dominant travel direction.

In Relation Operations (Intrinsics)

VEX

relpointbbox works, but intrinsics give cleaner, more predictable access to exact bounding box bounds — no node-level bbox weirdness. Great for height-based attribute distribution.

Point Wrangle — height ramp via bounding box intrinsics
float bounds[]    = detailintrinsic(0, "bounds");
float bounds_ymin = bounds[2];
float bounds_ymax = bounds[3];

vector spread = fit(v@P, bounds_ymin, bounds_ymax, 0, 1);
float ramp    = chramp("ramp", spread[1]);
@Cd.r         = ramp;   // red channel driven by Y height
detailintrinsic(0, "bounds") returns [xmin, xmax, ymin, ymax, zmin, zmax]. Primitive intrinsics follow the same pattern — check the VEX docs for the full list.

Calculate Radius

VEX Geometry

Calculates the radius of a single wire/ring by summing edge lengths and dividing by 2π. Best used inside a node-based for-each loop with Connectivity upstream so each piece is processed independently.

Detail Wrangle — radius from edge lengths
f[]@length;

for (int i = 0; i < @numpt - 1; i++) {
    vector pt1    = point(0, "P", i);
    vector pt2    = point(0, "P", i + 1);
    float measure = distance(pt1, pt2);
    push(@length, measure);
}

f@radius = (sum(@length)) / (2 * $PI);
Only gives meaningful results when the input is a single ring. Use Connectivity + a for-each loop to isolate individual rings before running this code.

Find Matchmaker

VEX

Two handy selection patterns: slice-based point selection from prim endpoints, and wildcard path matching — useful when node-based group expressions aren't flexible enough.

Prim Wrangle — select last/first two points per prim
i[]@last  = primpoints(0, @primnum)[-2:];
i[]@first = primpoints(0, @primnum)[:2];

if (find(i@last, @ptnum) == 0 || find(i@last, @ptnum) == 1) {
    v@Cd = 0;
}
Point Wrangle — wildcard path matching
if (match("*some_path*", prim(0, "path", @primnum)) == 1) {
    // do something
}

Custom Deformer (Solver-Based)

VEX CharFX Simulation

When a shot starts mid-collision and preroll isn't an option, this solver wrangle pushes geometry out of a collider SDF to clear space before simulation begins. A Switch upstream feeds first-frame geometry on frame 1, previous frame otherwise.

Solver Wrangle — SDF-based push
float offset        = chf("offset");
vector sample       = @P + @N * offset;
float dist          = volumesample(1, 0, @P);
vector distgradient = volumegradient(1, 0, @P);

f@intersect = 1;
float multiply = chf("multiply");
@P = (@P + (distgradient * multiply));

v@velocity = point(3, "v",     @ptnum);
v@speed    = point(3, "speed", @ptnum);

@P = @P + (normalize(v@velocity)) * v@speed / 24;

Variation: push based on distance from a single control point rather than an SDF volume.

Solver Wrangle — distance-from-point push
v@P2         = point(1, "P", 0);
float dist   = distance(@P, v@P2);
float ratio  = chf("ratio");
float min    = chf("min");
float max    = chf("max");
float vel    = point(2, "v",     @ptnum);
float speed  = point(2, "speed", @ptnum);

@P = @P + (@P - v@P2) * -ratio * fit(dist, min, max, 1, 0);
@P = @P + normalize(vel) * speed / 24;
The / 24 converts per-frame velocity to per-second. Adjust to match your project FPS.

Join Node Alternate

VEX Geometry

Stitches separate line primitives into a continuous polyline regardless of sort order or UV direction. Run the Prim Wrangle first to cache per-prim point counts, then the Point Wrangle to build the connections.

1 — Prim Wrangle
int primpoints_count[] = primpoints(0, @primnum);
i@primpoints_count     = len(primpoints_count);
2 — Point Wrangle
int primpoints_count[];
int sum_of_points[];
int sum_of = 0;

for (int i = 0; i < @numprim; i++) {
    int sum = prim(0, "primpoints_count", i);
    push(sum_of_points, sum);
    push(primpoints_count, sum);

    int sum_of_points_last = sum_of_points[-1:][0];
    sum_of += sum_of_points_last;

    addprim(0, "polyline", sum_of - 1, sum_of);
}
Finish with a PolyPath SOP. No Reverse nodes needed — unsorted primitives are handled cleanly.

Hashmaps / Dictionaries

VEX

VEX dictionaries (d@) give you string-keyed lookup tables. Handy for shot-context switching when there's no pipeline metadata — build the map once, read the current shot code, and index into it to drive switches.

Detail Wrangle — shot-context switcher
d@hashmap = set(
    "shotname_1", 1,
    "shotname_2", 2
);

s@shot_ref      = chs("/obj/RefNode/ShotCode");
string shot_ref = s@shot_ref;

i@switcheroo = getcomp(d@hashmap, shot_ref, 0);
Feed i@switcheroo into a Switch SOP. Extend the dictionary with as many shots as needed — no keyframes, no per-shot branches.

Procedural Attribute Adjustment with HScript

HScript

Paste these expressions directly into parameter fields instead of keyframing. They produce clean 0–1 ramps and compound motions that work across many shots without touching the Graph Editor.

HScript — 0 → 1 ramp over shot length
fit(@Frame, $FSTART, $FEND, 0, 1)
HScript — rise-and-fall curve (peak at midpoint)
fit(@Frame, $FSTART, $FEND-120, 0, 1) + fit(@Frame, $FSTART+120, $FEND, 0, -1)
Use WolframAlpha to visualise more complex equations before committing them to a parameter. Works particularly well across many shots where keyframing becomes unmanageable.

Culling with toNDC

VEX Simulation

Culling off-screen geometry is one of the most effective ways to cut RAM during simulation. toNDC converts world positions to camera space — anything outside [0, 1] (with margin) is off-screen and safe to blast.

Point Wrangle — camera frustum cull
string camera = chsop("camera");
vector campos = toNDC(camera, @P - chv("offset"));

if (campos.x > 1.05 || campos.x < -0.05 ||
    campos.y > 1.05 || campos.y < -0.05) {
    @group_blah = 1;
}
Important: do not run this directly as a POP/Geo wrangle inside a solver — use a SOP Solver with a Gas Intermittent to fire it once per frame.

For pyro: combine vel and density with a VDB Combine intersection to clip vel to density bounds. This keeps pyro from simulating in invisible regions.

Bounding Box Volume from Packed Objects

VEX Geometry

Packed geometry doesn't have its volume calculated by default and the Measure SOP fails silently on it. Reading the intrinsic bounds directly gives a fast approximate volume — enough to drive "blast anything under N m³" logic.

Prim Wrangle — approximate AABB volume of packed objects
float bounds[] = primintrinsic(0, "bounds", 0);
f@vol = abs(bounds[0] - bounds[1])
      * abs(bounds[2] - bounds[3])
      * abs(bounds[4] - bounds[5]);
This is an AABB approximation. For millions of packed pieces it's fast enough to drive logic like "blast anything under 1 m³ before converting to SDF."

Wrangle Remap to Dynamic Min/Max

VEX

When you need to normalize a value across all points but the range changes every frame, compute min/max dynamically in a Detail Wrangle rather than baking it with an Attribute Promote.

Detail Wrangle — per-frame dynamic min/max
float value[];

for (int i = 0; i < @numpt - 1; i++) {
    float pt_value = point(0, "some_value", i);
    append(value, pt_value);
}

f@value_min = min(value);
f@value_max = max(value);
// f[]@value = value;  // uncomment to expose the full array

Heightfield Wrangle (Mask & Height Conditionals)

VEX Simulation

Heightfields are volumes under the hood, so a Volume Wrangle reaches them when the built-in Heightfield nodes fall short. This example samples neighbouring masks and zeroes out regions by height range.

Volume Wrangle — sample masks / height filter
@mask -= volumesample(1, "mask", @P);

if (@mask < 0.001) {
    @mask += volumesample(2, "mask", @P);
}

if (@height < 50 || @height > 25) {
    @mask = 0;
}
Heightfields proved very effective as simulation colliders. They require hacking the solver to accept a Terrain Object input, but the performance gains versus surface collisions are significant.

Volume Operations Based on Variables

VEX

Volume Wrangles work much like Point Wrangles — distance functions, nearpoint, and chramp all apply. This pattern culls density by distance falloff from two separate point sources.

Volume Wrangle — density falloff from two point sources
// Falloff from a single point
float dist       = distance(@P, point(1, "P", 0));
float dist_remap = fit(dist, 1000, 0, 1, 0);
float ramped     = chramp("remap_this", dist_remap);
@density        *= ramped;

// Falloff from nearest point in a cloud
int near          = nearpoint(2, @P, 1000);
float dist2       = distance(@P, point(2, "P", near));
float dist_remap2 = fit(dist2, 1000, 0, 1, 0);
float ramped2     = chramp("remap_ya", dist_remap2);
@density          = ramped2;
When nearpoint gives unexpected distances, check that you're sampling from the correct context. Convert reference geo to points first if needed.

Aggregate Strings & Find Them Later

VEX

Collect all name attributes present at a given frame into a Detail string array, then filter out any geo whose name wasn't present at that reference frame — preventing ghost pieces, moblur blowouts, and unpredictable point counts.

Detail Wrangle — aggregate names (TimeShift to end of shot)
s[]@names;

for (int i = 0; i < @numpt - 1; i++) {
    string nam = point(0, "name", i);
    append(@names, nam);
}
Point Wrangle — remove points not in the reference list
s[]@names = detail(1, "names");

if (find(@names, @name) < 0) {
    // name not found — keep it
} else {
    removepoint(0, @ptnum, 1);
}
Especially useful in RBD setups where objects appear or disappear mid-sim and need consistent naming for motion blur to stay sane.

Playing Back .EXR Sequences with MPV

Tools

MPV is a free, highly configurable media player that plays back EXR image sequences directly from the command line — no Nuke licence or heavy DCC required.

CMD — play an EXR sequence
cd Desktop/mpv
mpv "mf://C:\Users\VeryCoolUser\Desktop\Project\render\AmazingRender.%04d.exr" --demuxer-lavf-format=image2
%04d matches four-digit zero-padded frame numbers (e.g. 0001, 0042). Adjust the padding to match your render output format.

Audio to Spectrum Conversion (FFmpeg)

Tools

Convert stereo audio into per-frame greyscale spectrum images readable in Houdini as a texture — no CHOPs required. Output is 1×1024 px strips per frame, one folder per channel.

FFmpeg — audio → spectrum image sequence
ffmpeg -i "C:\Users\YoPath\Music\YoSong.mp3" \
  -filter_complex \
    "[0:a]channelsplit=channel_layout=stereo[L][R]; \
     [L]showspectrum=s=1x1024:scale=lin:start=0:stop=20000:slide=replace:color=intensity,format=gray16be,fps=24[left]; \
     [R]showspectrum=s=1x1024:scale=lin:start=0:stop=20000:slide=replace:color=intensity,format=gray16be,fps=24[right]" \
  -map "[left]"  -fps_mode passthrough left\frame_%06d.png \
  -map "[right]" -fps_mode passthrough right\frame_%06d.png
Creates left/ and right/ folders of 1×1024 greyscale PNGs at 24 fps. Read them back in Houdini with a File SOP or as a texture inside a Volume Wrangle.
